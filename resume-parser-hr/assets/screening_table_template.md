说明：

- **匹配分**是**唯一的底层分数**（0-100）：四维平滑加权——经验匹配度 + 学历 + 履历稳定性 + 置信度，再减降权/P1/P2 软扣分。由脚本算好，**不要自己改算**。
- **推荐等级**由匹配分**单调推导**，二者永远一致（不会出现「高分低等级」）：
  - `🟢 强推荐` = 匹配分 ≥ 门槛（默认 75，`--pass-threshold` 或 `HR_PASS_THRESHOLD` 可调）
  - `🟡 待审核` = 门槛-10 ≤ 匹配分 < 门槛（默认 65–75）
  - `🟠 谨慎` = 门槛-20 ≤ 匹配分 < 门槛-10（默认 55–65）
  - `🔴 不推荐` = 匹配分 < 门槛-20（默认 < 55）
- **⚠️ 徽标**：候选人存在 P0 数据红旗（时间倒置、同期多份全职等）时，在推荐等级后追加 `⚠️`，并在**简要分析**里用粗体写出具体 P0。⚠️ **只提示、不改变分档**——是否录用由 HR 结合红旗判断。
- **排序**：按匹配分从高到低（等级已随分数单调，直接排即可）；前三名用 🥇🥈🥉。
- **简要分析**（最后一列）：一句话说清候选人的岗位背景 + 相关经验年限 + 学历 + 最匹配点/首要风险（+ P0 粗体）。这就是给 HR 的分析结论——**不要在表格外再单独列一段「简要分析/简要说明」，分析写在这一列即可**。需要更细的解析置信度/命中关键词/不匹配项/人工复核原因，用 `--output result.csv` 导出查看。
