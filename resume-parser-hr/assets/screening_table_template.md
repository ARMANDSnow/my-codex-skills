说明：
- **综合评分**为 0-100 证据强度分（强/弱判据 + 相关经验 + 稳定性 − 降权/异常扣分），由脚本算好，**不要自己改算**。
- **状态**：`✅ 通过` = 评分 ≥ 免复核阈值（默认 75，可用 `--pass-threshold` 或环境变量 `HR_PASS_THRESHOLD` 调整）且无 P0 异常，可直接通过、免复核，HR 自愿承担风险；`✅⚠️` = 评分达标但存在 P0 异常，仍算通过，但 **P0高亮列** 已用粗体写出具体内容，务必关注；`待筛选` = 评分未达阈值，进入人工筛选。
- 解析置信度低于 0.6 时即便评分达标也会被降为 `待筛选`（数据未解析可靠，不盲目免复核）。
- 排序优先级：状态（✅ 通过 → ✅⚠️ → 待筛选），其次综合评分由高到低。
- 人工复核原因优先处理 P0/P1 异常、低解析置信度、图片简历、时间线冲突和 JD 输入问题。
