# 异常校验与稳定性规则

## 异常等级

- P0：拦截类，必须人工复核，不得自动强推荐。
- P1：降权类，降低推荐等级或进入待审核。
- P2：提示类，增加追问项，不直接淘汰。

年龄、出生年份等敏感属性不得用于评分或排序。若候选人主动提供年龄且与履历时间线明显矛盾，只能输出“人工复核提示（不参与评分）”。

## 一期异常

| 异常 | 条件 | 等级 |
|---|---|---|
| 履历时间线提示 | 候选人自述年龄与正式工龄明显不一致 | P2（不参与评分） |
| 工龄开始时间提示 | 按候选人自述年龄推算，正式工作开始时间异常偏早 | P2（不参与评分） |
| 时间倒置 | 开始日期晚于结束日期 | P0 |
| 同期多份全职 | 同期 ≥2 段正式工作 | P0 |
| 同期多段经历 | 同期 ≥3 段任何类型经历 | P0 |
| 全职实习重叠 | 正式工作与实习重叠 | P1 |
| 经历可信度低 | 可信度分 < 0.4 | P1 |
| 岗位跳跃异常 | 最近三段岗位相近度多数 < 0.3 | P1 |
| 近期频繁跳槽 | 最近 2 年 3 次变动或 3 段短任职 | P1 |
| 学历层级缺失 | 教育背景缺少学历 | P1 |
| 简历信息可能未更新 | 简历最新年份距今 ≥ 约 2 年（在职简历放宽到 ≥ 3 年） | P2 |
| 时间缺失/项目重叠/学校缺失 | 信息不足但可补充 | P2 |

## 简历时效（信息是否陈旧）

判断简历最新信息距今多久，提醒可能未更新；只提示，不参与评分、不淘汰。

- 取简历文本中出现的最新四位年份（`latest_year_in_text`，排除手机号/工号中的数字片段，忽略未来年份）。
- 普通简历（无“至今/在职”）：最新年份距今 **≥ 约 2 年** 即提醒（如今年 2026、简历最新到 2023）。
- 在职简历（出现“至今/现在/在职”等）：临界值放宽到 **≥ 3 年**，避免误伤长期在职者；触发时改为提醒“确认是否仍在职及最新经历”。
- 用文本最新年份而非经历结束日期判断，避免“至今”被解析成当前月份后掩盖陈旧简历。
- 输出落在 `resume_recency`（`latest_year_in_text`、`years_since_latest`、`has_ongoing`、`is_stale`），并在 `is_stale=true` 时写入 P2 异常“简历信息可能未更新”。

## 稳定性分

履历稳定分 = 100 * (
0.35 * MIN(平均在岗月数, 42) / 42
+ 0.15 * MIN(中位在岗月数, 36) / 36
+ 0.20 * (1 - 短任职占比12)
+ 0.10 * (1 - 严重短任职占比6)
+ 0.20 * MIN(最近2段平均在岗月数 / MAX(更早经历平均在岗月数, 1), 1)
)

标签：

- 优秀：≥80
- 良好：60-79
- 一般：40-59
- 较差：<40

## Gap 分

Gap 分 = 100 * (
0.50 * (1 - MIN(近5年总Gap月数, 12) / 12)
+ 0.30 * (1 - MIN(最长Gap月数, 6) / 6)
+ 0.20 * Gap解释分
)

Gap 解释分：

- 有备考、进修、创业、项目、照顾家庭、生育、病休等明确说明：0.85
- 无说明：0.35
- 无 Gap：1.0

标签：

- 优秀：≥80
- 良好：70-79
- 一般：60-69
- 较差：<60

### Gap 不止给分，必须具体说明

仅输出 Gap 分不够，必须同时给出可读的空窗说明，便于 HR 直接追问：

- `gap_details`：逐段空窗，含前后公司、起止年月（`from_date → to_date`）、空窗月数、是否近 5 年内、是否有说明（命中备考/考研/进修/创业/项目/照顾家庭/生育/病休等关键词记为“有说明”，并回填命中的 `reason`）。
- `gap_summary`：一句话汇总空窗段数、累计月数、最长月数、近 5 年月数，以及“有说明 / 无说明”段数。
- 无空窗时显式说明“无明显空窗”，不要只丢一个分数或留空。
