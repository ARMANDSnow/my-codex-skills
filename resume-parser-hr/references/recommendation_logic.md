# 推荐逻辑（证据强度评分 + 免复核状态）

## 决策原则

不要只看关键词；优先看证据强度、最近相关性、异常等级、稳定性和 Gap。
输出收敛为**一个 0-100 综合评分**与**一个状态标签**，不再使用「强推荐/待审核/暂不推荐」三档。

## 0-100 证据强度综合评分

由脚本 `recommendation_engine.calculate_evidence_score()` 算好（**不要手算**），满分 100：

```
综合评分 = BASE + WEAK + RELEVANT + STABILITY  -  DOWNGRADE - ANOMALY - P0_DAMPEN
- BASE       强判据（学历/经验/相关性/稳定性）全部满足给 45，否则 0
- WEAK       每项弱判据 +3.6，最多 5 项（+18）
- RELEVANT   可信相关经验：满 12 个月 +12；每段高可信相关经历 +4，最多 2 段（合计 +20）
- STABILITY  履历稳定分/100 × 10 + Gap 分/100 × 7（最多 +17）
- DOWNGRADE  每个降权因子 -6，最多 3 项（-18）
- ANOMALY    每个 P1 -8，每个 P2 -3（仅计参与评分的异常）
- P0_DAMPEN  每个 P0 -6，最多 -12（软抑制，不硬拦截）
```

BASE 只看**实质性**强判据，P0 与「P1 数量超标」不计入 BASE：P0 改由软抑制 + `✅⚠️` 状态体现，P1 由 ANOMALY 扣分体现。否则带 P0 者分数归零，`✅⚠️` 兜底态将无法触发。

## 状态标签与免复核阈值

阈值默认 **75**，可用 `--pass-threshold` 或环境变量 `HR_PASS_THRESHOLD` 调整（CLI > 环境变量 > 默认）。

- **`✅ 通过`**：综合评分 ≥ 阈值 且无 P0 异常 → 可直接通过、免复核，HR 自愿承担风险。
- **`✅⚠️`**：综合评分 ≥ 阈值 但存在 P0 异常 → 仍算通过，但必须看 P0 高亮（脚本在 `recommendation.p0_remark` 给出具体内容，展示时加粗）。
- **`待筛选`**：综合评分 < 阈值 → 进入人工筛选。
- 额外：解析置信度 < 0.6 时，即便评分达标也降为 `待筛选`（数据未解析可靠不盲目免复核）。

## 实质性强判据（决定 BASE 是否为 45）

1. 学历达到 JD 最低要求；JD 写学历不限则不拦截。
2. 中高可信销售相关经验达到 JD 要求；无 JD 时默认至少 6 个月。
3. 最近一段经历与销售相关，相似度至少 0.5。
4. 销售经历不能只是一行岗位名；需具备职责、客户对象、结果/指标或完整时长。
5. 履历稳定分 ≥ 60，Gap 分 ≥ 70。

## 弱判据

- 客户沟通、谈判、维护、服务经验。
- 简历字段完整，解析置信度 ≥ 0.75。
- 稳定性尚可：履历稳定分 ≥ 60 且 Gap 分 ≥ 70。
- 有业务结果指标：转化、成交、销售额、回款、达成率、排名、单量等。
- 行业或职能可迁移：客服、市场、BD、渠道等。

## 降权因子

- 只有一段低可信相关经历。
- 相关经历久远，最近两段不相关。
- 时间线异常或重叠。
- 近期跳槽频率偏高。
- 履历稳定分 < 50 或 Gap 分 < 60。

## 一行经历误强推修正规则

每段经历计算 `credibility_score`：

- 有岗位名：0.2
- 有 10 字以上职责描述：0.3
- 有结果/指标：0.3
- 有客户对象：0.1
- 有完整时长：0.1

低可信 `<0.4`，中可信 `0.4-0.69`，高可信 `>=0.7`。

若销售相关经历低可信，不计入 BASE 的实质强判据，综合评分难以达到免复核阈值。
