# 推荐逻辑（匹配分 + 单调推荐等级）

## 决策原则

不要只看关键词；优先看证据强度、最近相关性、异常等级、稳定性和 Gap。
输出收敛为**一个 0-100「匹配分」**，**推荐等级由匹配分单调推导**（分越高等级越高，二者永不矛盾）。

## 0-100「匹配分」：四维平滑加权

由脚本 `recommendation_engine.calculate_evidence_score()` 算好（**不要手算**）。与旧「BASE 全有或全无」不同：
**缺一项判据只按比例扣分，不再直接归零**，避免真实资深候选人（如描述单薄的多年销售）被压到低分。

```
匹配分 = D1 经验 + D2 学历 + D3 稳定 + D4 置信度  -  DOWNGRADE - ANOMALY   （0-100）
- D1 经验(权重40)   0.55×相关月数(满30个月封顶) + 0.25×最近一段相关度 + 0.20×高可信段数(满2段)
- D2 学历(权重15)   候选人学历 / JD 要求（无要求给满分，差一级给部分分）
- D3 稳定(权重25)   履历稳定分/100 × 0.6 + Gap 分/100 × 0.4
- D4 置信度(权重20) 解析置信度 × 0.75 + 相关经历可信占比 × 0.25
- DOWNGRADE         每个降权因子 -3，封顶 -9
- ANOMALY           每个 P1 -6（封顶 -18），每个 P2 -3（封顶 -9）；仅计参与评分的异常
```

**P0 不进匹配分**：P0（时间倒置、同期多份全职、简历可能未更新等）是数据红旗，只由推荐等级追加 `⚠️` 徽标呈现，**不扣分、不改变分档**——是否录用交给 HR 结合红旗判断。解析置信度已并入 D4，低置信度自然拉低匹配分，不再单独硬降级。

## 推荐等级：匹配分单调分档

强推荐门槛默认 **75**，可用 `--pass-threshold` 或环境变量 `HR_PASS_THRESHOLD` 调整（CLI > 环境变量 > 默认）。
由 `recommendation_engine.fit_tier()` 推导，同一匹配分永远对应同一等级：

| 匹配分 | 推荐等级 | 色点 |
| --- | --- | --- |
| ≥ 门槛（默认 75） | 强推荐 | 🟢 |
| 门槛-10 ~ 门槛（默认 65–74） | 待审核 | 🟡 |
| 门槛-20 ~ 门槛-10（默认 55–64） | 谨慎 | 🟠 |
| < 门槛-20（默认 < 55） | 不推荐 | 🔴 |

存在 P0 时等级后追加 `⚠️`（脚本在 `recommendation.p0_remark` 给出具体内容，展示时加粗），**不改变分档**。

## 实质强判据 / 相关性口径（用于理由文案与经验维度）

以下判据仍由脚本判定，用于生成 `reason`、喂 D1/D2 维度，但**缺项只按比例扣分、不再一票归零**：

1. 学历达到 JD 最低要求；JD 写学历不限则不拦截。
2. 中高可信销售相关经验达到 JD 要求；无 JD 时默认至少 6 个月。
3. 最近一段经历与销售相关，相似度至少 0.5。
4. 销售经历不能只是一行岗位名；需具备职责、客户对象、结果/指标或完整时长。
5. 履历稳定分 ≥ 60，Gap 分 ≥ 70。

## 弱判据

- 客户沟通、谈判、维护、服务经验。
- 简历字段完整，解析置信度 ≥ 0.75。
- 稳定性尚可：履历稳定分 ≥ 60 且 Gap 分 ≥ 70。
- 有业务结果指标：转化、成交、销售额、回款、达成率、排名、单量等。
- 行业或职能可迁移：客服、市场、BD、渠道等。

## 降权因子

- 只有一段低可信相关经历。
- 相关经历久远，最近两段不相关。
- 时间线异常或重叠。
- 近期跳槽频率偏高。
- 履历稳定分 < 50 或 Gap 分 < 60。

## 一行经历误强推修正规则

每段经历计算 `credibility_score`：

- 有岗位名：0.2
- 有 10 字以上职责描述：0.3
- 有结果/指标：0.3
- 有客户对象：0.1
- 有完整时长：0.1

低可信 `<0.4`，中可信 `0.4-0.69`，高可信 `>=0.7`。

若销售相关经历低可信，会触发 P1（低可信经历）软扣分、并拉低 D4 置信度维度与高可信段数（D1），
使匹配分下降、推荐等级随之降低——但不会像旧模型那样一票归零，真实长年限经验仍按比例得分。
