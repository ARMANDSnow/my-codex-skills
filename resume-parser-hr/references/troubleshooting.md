# 排障：打分异常 / 多年销售被判“无销售经验”

当 HR 反馈「明明有多年销售经验，却被判无经验、分数异常低」时，按下面三步走。
**先别急着怀疑评分逻辑——多数情况是拿到的文件不是最新版，或某份简历踩到解析盲区。**

> 一句话给同事/她的 agent：把这个文件夹里的 `scripts/diagnose_resume.py` 跑一下，把输出贴回来。

## 第 0 步：确认在 skill 目录里

```bash
cd <你的 my-codex-skills 仓库>/resume-parser-hr      # 或 ~/.codex/skills/resume-parser-hr
```

## 第 1 步：一键自检（版本 + 文件完整性 + 引擎是否正常）

```bash
python3 scripts/diagnose_resume.py
```

看最后的「结论」：

- **✅ 版本为最新（2.2）、文件完整、引擎正常** → 文件没问题，跳到第 3 步诊断那份具体简历。
- **❌ 检测到问题** → 按提示更新（见第 2 步）。常见两种：
  - `version` 不是 2.2，或 C 段把多年销售判成无经验 → **是旧版/未修复版**。
  - 某些文件 sha256 不一致 / 缺失 → **文件被改动或损坏**。

自检三段含义：
- **A 版本/完整性**：安装版本号、git HEAD、7 个核心文件的 sha256 比对。
- **B 引擎自测**：内置销售样例必须判为“有经验、≥75 分”。
- **C 已知良好回归**：一份“多年销售但描述单薄”的简历，必须算出真实年限（不得出现“无经验/0个月”）。

## 第 2 步：更新到最新版

```bash
cd <你的 my-codex-skills 仓库>
git pull
./install.sh          # 重新拷贝到 ~/.codex/skills/
```

然后重跑第 1 步，确认结论变成 ✅。

## 第 3 步：诊断具体那份“被误判”的简历

```bash
python3 scripts/diagnose_resume.py 那份简历.txt --jd JD.txt --job-title 销售
# PDF/Word 也支持；若缺依赖报错，可先把简历另存为 .txt 再诊断
```

D 段会逐段打印每段经历的 `type / 原岗位 / 标准岗位 / 起止 / 月数 / 可信度 / 描述长度`，外加
`relevant_months`、`status`、`reason`、未满足强判据，并在「定位」给出一句话结论：

- **没解析出任何经历** → 日期格式/版面问题（表格版面、英文月份、特殊分隔符等）。把简历发回排查。
- **经历在、但标准岗位都不算销售** → 岗位名词典没覆盖该写法，把原岗位名发回补词典。
- **相关经验已正确计入** → 分数偏低多半是描述单薄 / 稳定性 / 异常项导致，看 `reason` 和未满足项即可解释。

把 D 段输出整段贴回来，就能精准定位，不用猜。

---

### 附：评分口径速记（避免误读“低分=系统坏了”）

- 评分是 **0-100「匹配分」**（四维平滑加权），推荐等级由它单调推导：**≥门槛(默认75)=强推荐，65+=待审核，55+=谨慎**；**60 来分、待审核 ≠ 无经验**，而是“经历真实但证据单薄，建议人工看一眼”。
- “相关经验月数”按**相关销售岗 + 正式/实习 + 有效时间**计入（区间并集，不重复累加）；
  只写“公司+岗位+时间”、没写量化业绩的，**仍计入年限**，只是可信度偏低、需人工看。
- 真正会判“缺少强证据/低可信”的，是**又短又没有任何职责/业绩/客户信息**的经历。
